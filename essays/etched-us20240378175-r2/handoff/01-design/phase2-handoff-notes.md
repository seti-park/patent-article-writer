## (d) Traps to avoid

**Claim-scope traps (restating the invention-summary Claim scope map — sought/open/pinned
vocabulary; NO locked class exists in this edition):**

- DO use application-era language for every claim statement: "the application claims",
  "Etched is seeking", "as drafted". DON'T ever write "the patent locks / requires /
  fences / protects" — there is no patent; there are no enforceable claims; nothing here
  can be infringed unless (until) it grants. No enforceability language anywhere.
- DO treat these as SOUGHT-required (claim text as drafted): claim 1's plurality-of-ICs +
  chip-to-chip connections forming a combined array; claim 15's weight-storing memory chips
  coupled to the top row; claim 26's grid-like pattern; claim 39's separate memory device
  with EACH channel hardwired to one or more columns WITHOUT ANY switching element; claim
  13's arrays-do-not-communicate-with-local-memory (negative limitation, sought). DON'T
  present any of them as owned or settled.
- DON'T present description preferences as claim content (the map's "leaves open" column):
  UCIe, 32 GT/s / 16-64 lanes, interposer vs stacking, unidirectional-vertical /
  bidirectional-horizontal split (deps 3/4/16-18/27-29 only), HBM type (deps 7/19/32/41
  only), weight constancy (`[0035]` is description), the preset loop `[0027]`, the 128×128
  ceiling `[0018]`, hundreds-of-GBs links `[0032]`, 1 TB/s `[0040]`, and ALL of FIG. 7
  (batching, three passes, GeLU pre-feed, layer-norm stall, 98% utilization). These are the

- PINS: there are NO pinned "about X" point limitations in any claim. DON'T attach

- DO note (if §3 needs the honesty beat) that claim 39 does not require multi-chip and
  claim 1 does not require memory chips at all (`[0037]`) — the two ideas are claimed
