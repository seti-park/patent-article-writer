# Figure Rationale

One block per selected figure (same set as figure-selection.md): why it earns its place (purpose) and what it should do to the reader (intended effect). Phase 2 reads this alongside figure-selection.md when planning placement + caption.

## FIG. 1B — The 8-high die stack (cover / header)

- **Purpose**: Shows the claimed memory die stack (111) — dies (112) with aligned databases (113) and TSVs (114) — stacked eight-and-beyond over an optional base die (115). It is the literal picture of claim 1's "memory die stack" and `[0023]`'s "8-high and beyond."
- **Intended effect**: Instant feed force — the reader sees "a new kind of tall, custom memory tower" before reading a word, and the header primes the whole spine (a stack aimed at HBM4's footprint). Chosen as cover for legibility at 5:2 and a ≤ 6-numeral caption. (caption_role: header_composite — the fuller caption; keep it to ~6 numerals for SURF-003.)

## FIG. 1A — Logic die beside the memory stack (the thesis image)

- **Purpose**: Shows package 100 with the logic die (106) and the HBM stack (104, dies 105) sitting on one interposer (102) over the package substrate (101). It is the only figure that puts logic and memory on a single package in one frame.
- **Intended effect**: Grounds the strategic reframe visually — "logic + memory on one flow" is the whole point of §4-who-can-make-it. When the prose argues that a back-end cell lets a logic-plus-packaging line carry HBM-class memory, this figure is the picture the reader already has in front of them. (caption_role: body_figure_carries_unique_info — a medium caption naming the logic die vs the memory stack; do not exceed the co-package point.)

## FIG. 1F — The backend transistor layers (the claimed novelty)

- **Purpose**: Cross-section 150 with exploded inset 150A, showing the stacked layers explicitly labeled "TRANSISTOR" and "GM" as thin-film-transistor databases (151/153) separated by TSV/HBI regions (152/154) across wafers (155-157) — the direct depiction of `[0031]` and the meaning of the claim word "backend."
- **Intended effect**: Makes "the DRAM cell lives in the back-end" concrete rather than abstract — the reader sees transistor layers stacked in the metal, which is exactly the fact the spine turns into "who can make HBM." Also the honest place to note the capacitor is relocated, not removed. (caption_role: body_figure_carries_unique_info — a medium caption; gloss "backend/thin-film transistor" once, next to the figure.)

## FIG. 1G — The building block and TSV gutters (match-HBM4 architecture)

- **Purpose**: Isometric schematic of the basic memory building block (160): eight sub-channels (161A-161H) separated by four TSV gutters (162-165), the datablock unit the patent architects "with the goal of matching HBM4's footprint" `[0033]`, `[0034]`.
- **Intended effect**: Carries the §3 mechanism so the prose can stay light — the reader sees how the die is partitioned into channels/sub-channels and how data drops through TSV gutters, then the text can move to the stakes. Strong, dark 3D grid; visually the most striking body figure. (caption_role: body_figure_carries_unique_info — name the sub-channels + TSV gutters; the "match HBM4's footprint" goal belongs in prose, labeled as the patent's goal, not a measured result.)
